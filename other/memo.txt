-----------uwsgi-----------1. 下载：http://projects.unbit.it/downloads/uwsgi-0.9.6.6.tar.gz2. 编译uwsgi   由于uwsgi需要libxml2库的支持 需要将libxml2先安装上 apt-get install python-dev libxml2-dev	tar zxvf uwsgi-0.9.6.6.tar.gz	cd uwsgi-0.9.6.6	make -f Makefile3. 编译成功后就会在当前目录下生成一个uwsgi的可执行文件	cp uwsgi /usr/sbin/uwsgi4. 配置app	添加运行uwsgi的用户	adduser --system --no-create-home --disabled-login --disabled-password --group uwsgi	将uwsgi的配置写在了一个xml文件中，文件名uswgi.xml	<uwsgi>	  <socket>10.2.74.100:3031</socket><!-- 必须是具体ip -->	  <master>true</master>	  <pidfile>/var/lib/nginx/uwsgi/uwsgi.pid</pidfile>	  <gid>uwsgi</gid>	  <uid>uwsgi</uid>	  <processes>4</processes>	  <pythonpath>/var/salon</pythonpath> <!--Django应用目录-->	  <module>wsgi</module> <!--Django应用目录下包含的wsgi.py文件名，-->	  <profiler>true</profiler>	  <memory-report>true</memory-report>	  <enable-threads>true</enable-threads>	  <logdate>true</logdate>	  <limit-as>512</limit-as>	</uwsgi>		编写wsgi.py文件内容如下：	import sys	import os	sys.path.append('/var/salon')	os.environ['DJANGO_SETTINGS_MODULE']='fedclub.settings' #项目名.settings	import django.core.handlers.wsgi	application = django.core.handlers.wsgi.WSGIHandler() 	5. 配置nginx	配置文件在/etc/nginx/目录	配合uwsgi的设置，nginx.conf必要设置如下：	location / {		 root /var/salon;		 uwsgi_pass 10.2.74.100:3031;  # 此处也必须为具体ip		 include uwsgi_params;	}6. 运行	sudo uwsgi -x uwsgi.xml	将此命令写入runserver.sh中，放置在/var/salon目录下，进入此目录：	screen	sudo sh runserver.sh	exit	大功告成	-----------nginx/mysql-------------------1. 重启nginx： nginx -s reload2. 配置mysql，允许远程登录：	a. 修改配置文件		vim /etc/mysql/my.cnf		# Instead of skip-networking the default is now to listen only on		# localhost which is more compatible and is not less secure.		# bind-address           = 127.0.0.1  <---注释掉这一行就可以远程登录了	b. 创建远程登录用户： 		GRANT ALL PRIVILEGES ON *.* TO 'monty'@'%' 			IDENTIFIED BY 'some_pass' WITH GRANT OPTION;	c. 乱码问题：		修改mysql配置文件		vim /etc/mysql/my.cnf		    default-character-set=utf8		    init_connect='SET NAMES utf8'	d. 重启mysql		http://yangsiwei.com/restart-mysql-in-linux		$mysql_dir/bin/mysqladmin -uroot -p shutdown		$mysql_dir/bin/mysqld_safe &		或 终极招数： killall mysql		from wentao:			/etc/init.d/mysql start* 			/etc/init.d/mysql stop* 			/etc/init.d/mysql restart		其他：	mysql数据备份：		mysqldump -u wentaome -h host -p --databases dbname >/dir/file_name	mysql数据恢复：		mysql -u root -p dbname < file_name.dump
